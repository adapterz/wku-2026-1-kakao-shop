# 🛠️ 트러블슈팅 로그 (에러 해결 과정)

## [2026-07-01] GitHub Push 시 403 Permission Denied 에러 발생
- **증상**: 터미널에서 `git push`를 수행할 때 `remote: Permission to ... denied to [사용자명]` 에러가 발생하며 푸시가 거부됨.
- **원인**: 
    1. 컴퓨터에 저장된 기존 GitHub 계정 정보(Credential)와 현재 팀 레포지토리에 초대된 계정이 일치하지 않음.
    2. 또는 Windows 자격 증명 관리자에 예전 개인 계정 정보가 남아있어 권한 충돌이 발생함.
- **해결**:
    1. Windows [자격 증명 관리자] 실행.
    2. [Windows 자격 증명] 탭에서 `git:https://github.com` 관련 항목을 모두 삭제.
    3. 다시 터미널에서 `git push` 수행 시 나타나는 팝업(또는 터미널 입력창)에 팀 레포지토리에 초대받은 정확한 계정 정보를 입력하여 다시 인증.
- **배운 점**: Git은 컴퓨터 내의 자격 증명을 우선순위로 사용하므로, 팀 프로젝트 시작 시 반드시 깃허브 계정 인증 정보를 새로고침해야 함을 깨달음.
