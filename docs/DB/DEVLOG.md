## 💡 트러블슈팅 (Troubleshooting)
* **이슈:** Git Bash에서 `mysql` 명령어 입력 시 `command not found` 에러 발생.
* **원인:** 환경 변수(Path)에 MySQL 실행 파일이 위치한 정확한 `\bin` 폴더 경로가 누락됨.
* **해결:** Windows 시스템 환경 변수 편집에서 `C:\Program Files\MySQL\MySQL Server 8.0\bin` 경로를 정확히 추가한 뒤, 터미널을 완전히 재시작하여 해결.

### 2. 고민 및 해결 (트러블슈팅)
- **MySQL 백그라운드 프로세스 중단 에러**: root 계정의 플러그인 비밀번호 변경 쿼리를 실행하던 도중 키보드 오입력(`Ctrl + Z`)으로 인해 MySQL 프로세스가 백그라운드로 튕겨 나가며 중단(`[1]+ Stopped`)되는 현상이 발생함.
- **해결 방식**: 리눅스 작업 제어 메커니즘을 이해하고, 우분투 터미널 상태에서 `sudo mysql -u root` 명령어로 세션을 재진입하여 정상적인 쿼리 입력 환경을 확보함. 보안 검증을 거쳐 최종 마스터 패스워드(`gguo5362@`) 설정을 마친 후 `FLUSH PRIVILEGES;` 문으로 캐시를 갱신하여 커넥션 권한 문제를 해결함.
